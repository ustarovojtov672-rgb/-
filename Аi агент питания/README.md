# Аi агент питания

Локальное рабочее место для `pi-coding-agent`, который должен считать еду не одной догадкой по фото, а через агентский цикл: память, зрение, OCR, штрихкод, локальная база продуктов и интернет-поиск.

## Что уже подключено в приложении

- `/api/analyze-meal` вызывает серверный `nutrition-agent`.
- По умолчанию route использует `NUTRITION_AGENT_RUNTIME=pi` и запускает Pi через SDK.
- Агент получает текст, фото, профиль, цель, дневные ориентиры и последние приемы пищи.
- Агент использует память прошлых приемов и локальную базу продуктов из `src/lib/nutrition-agent`.
- Для неясных брендов, упаковок и ресторанов агенту доступен `codex_search`.
- UI показывает черновик расчета, объяснение агента, предположение по порции и факты перед сохранением.

## Логин в Pi

```bash
npm run nutrition-agent:login
```

В открывшемся Pi выполните:

```text
/login openai-codex
```

Токены сохраняются локально в `.pi-agent/` внутри этой папки и не попадают в git.

## Запуск агента вручную

```bash
npm run nutrition-agent:run -- "На фото упаковка творога, нужно посчитать 200 г"
```

Если аргумент не передан, будет использован `agent-prompt.md`.

## Роль этой папки

Эта папка фиксирует отдельного Pi-агента питания: его prompt, skill, auth-путь и сценарии запуска. Серверный код приложения использует этот workspace через `src/lib/nutrition-agent/pi-agent.ts`.

## Переключение runtime

```env
NUTRITION_AGENT_RUNTIME=pi
```

Такой режим использует Pi SDK и auth из `.pi-agent/auth.json`.

```env
NUTRITION_AGENT_RUNTIME=openai
```

Такой режим использует прямой OpenAI Responses API и требует `OPENAI_API_KEY`.
